## Problem Statement

中国大量40+中年人因行业结构性衰退（如房地产寒冬）而失业。他们有20年以上的行业经验，但这些经验在原行业中已失去价值。核心问题不是"没有技能"，而是：1）已有技能找不到变现渠道；2）长期失业导致信心崩塌；3）体力型零工（外卖/滴滴）无法匹配他们的能力水平。

典型案例：刘为，44岁，建材行业老兵，对建材价格和平替产品底价十分熟悉。行业崩塌后失业半年，送外卖和跑滴滴只能赚基本生活费，有老有小，信心接近谷底。

## Core Insight

创始人深度观察发现：

1. **真正的瓶颈是"不知道怎么卖自己"，不是"不知道自己知道什么"** — 刘为能当场说出瓷砖80块和25块的区别，他知道自己懂什么。
2. **但不能直接跳到"怎么卖"** — 用户需要循序渐进的心理重建过程。上来就谈获客和渠道，会让他们懵。
3. **产品应该像 gstack 的 skill 系统** — 结构化、一步一步、一个时间一个问题、每一步都有具体产出物。
4. **信心重建贯穿全程** — 不是附加功能，是核心机制。没有信心的用户做不成生意。

## Product Overview

**名称**：经验变现导航器 (Career Compass)
**定位**：AI变现合伙人 — 从"讲你的故事"到"赚到第一笔钱"
**目标用户**：35-55岁中国失业中年人，有10年以上行业经验，因行业衰退失业
**排除用户**：纯体力劳动者、完全丧失信心不愿行动的人

## Methodology: Case-first

核心方法论：**不是"先认识自己"，而是"先讲你的故事"。**

用户不会描述自己（"我的优势是什么"），但会描述具体事（"那次我帮客户省了3000块"）。系统从真实案例中提取可变现资产。

## User Journey: 4 Skills

### Skill 1: /认识自己（完整实现）

**方法**：Case-first + 证据导向问题

问题链（每个问题有选项 + 自由补充，用户可回退到任何步骤）：

1. 你在哪个行业干了多少年？
2. 过去10年，哪件事你做得比身边大多数同行都稳？
3. 讲一个你帮别人省钱或避坑的真实例子，越具体越好
4. 你知道哪些"行内人觉得正常，外行根本不知道"的信息？
5. 以前客户/同事最常因为什么来找你？
6. 你能判断什么东西"贵了、坑了、不值"？
7. 你有没有一类资源是别人短时间拿不到的：报价、渠道、工厂、流程、名单、经验？
8. 如果明天让你只靠经验赚到第一笔100元，你最可能卖什么帮助？

**输出**：资产清单（从案例中提取的可变现知识点 + 可用资源 + 信心评估）
**心理目标**："原来我懂这么多" — 信心 +1

### Skill 2: /规划路径（完整实现）

**方法**：约束筛选 + offer 生成

约束筛选问题（先筛选再推荐）：

1. 你最急的是几天内见到钱，还是先把口碑做起来？
2. 你愿不愿意直接和陌生人聊天/打电话？
3. 你更能接受卖服务、卖信息、还是帮人代办/撮合？
4. 每周能投入几小时？
5. 能接受首次免费换评价吗？
6. 你最不愿意做的事是什么？

**输出**：
- 首选 offer（具体的服务名称 + 目标客户 + 定价 + 交付形式）
- 为什么适合你
- 7天内第一步
- 如果无人回应，切换到哪个备选

**心理目标**："有人愿意为此付钱" — 方向感

### Skill 3: /开张行动（简化版）

**做**：
- 基于选定的 offer，生成首发文案（闲鱼/小红书/朋友圈）
- 基本的发布指导（发布时间、标签、图片建议）
- 文字生成 only，不做图片

**不做（验证后增强）**：
- 图片/视频生成
- 自动发布
- 多平台一键分发
- 数据追踪仪表盘

**心理目标**："我可以做到" — 行动力

### Skill 4: /卡住了（简化版）

**做**：
- 基本问题诊断（不知道怎么做 / 做了效果不好 / 技术困难 / 心理障碍）
- AI 给出针对性建议
- 解决后自动回到主线流程

**不做（验证后增强）**：
- 复杂的卡点分类
- 行业特定的解决方案库
- 人工导师接入

**心理目标**："问题可以解决" — 韧性

## Confidence Rebuilding System

### 三层证据结构

信心不能只靠 AI 鼓励。真正能修复信心的是三类证据：

1. **能力证据** — "你的案例确实帮人省过4000元"（从用户原话提取）
2. **市场证据** — "有人点击/私信/追问"（需要实际市场反馈）
3. **行为证据** — "你今天真的发出去了"（行为本身就是胜利）

### 信心机制

- **具体正向反馈** — 不是空泛的"你很棒"，而是"你的瓷砖知识能帮100平业主省5000元"
- **小胜利机制** — 每完成一步都有可见产出
- **负面情绪检测** — 检测"我不行""没什么用"等信号，共情回应 + 具体证据反驳
- **进度可视化** — 展示完成百分比和下一步
- **成功案例激励** — 类似背景的真实案例

### 防过度鼓励机制

- 有证据才强化
- 没证据就继续追问
- 不把普通经历硬包装成稀缺能力
- AI 不会对每个回答都夸，避免快速失真

## Technical Architecture

```
前端：Web 页面（React/Next.js）
  ├─ 对话区（当前 skill 的引导式对话 + 选项按钮）
  ├─ 产出物面板（资产清单 / offer / 文案）
  └─ 回退按钮（回到任何之前的步骤）

后端：Python + FastAPI
  ├─ Skill Engine
  │   ├─ SkillRegistry（注册4个skill）
  │   ├─ SkillRunner（执行当前skill的工作流）
  │   ├─ StateManager（用户状态 + 进度 + 回退）
  │   └─ ConfidenceEngine（三层证据 + 防过度鼓励）
  ├─ Skills/
  │   ├─ self_discovery/（Case-first 问题链 + 资产提取）
  │   ├─ plan_path/（约束筛选 + offer 生成）
  │   ├─ take_action/（文案生成 + 发布指导）[简化版]
  │   └─ troubleshoot/（问题诊断 + 建议）[简化版]
  └─ Content Generator
      └─ TextGenerator（DeepSeek API）

存储：SQLite
  ├─ 用户数据、进度、对话记录
  └─ 核心实验数据：用户故事 → 资产点 → offer → 发布结果

AI：DeepSeek API（中文优化，成本低）
图片：MVP 不做图片生成，验证后加
部署：本地先跑 → 验证后云部署
```

## Skill Workflow Definition

每个 skill 用 Python 模块定义：

```python
# skills/self_discovery/skill.py
class SelfDiscoverySkill:
    name = "认识自己"
    description = "引导用户梳理经验，发现可变现能力"
    order = 1

    steps = [
        Step(id="industry", question_prompt="industry",
             options=["建材/装修", "餐饮/食品", "零售/电商", "其他（请补充）"],
             allow_free_text=True),
        Step(id="proud_moment", question_prompt="proud_moment",
             allow_free_text=True,
             confidence_boost="evidence_replay"),
        # ... 更多步骤
    ]

    def generate_output(self, answers):
        return CapabilityMap(answers)
```

## Scalability Path

扩展路径（行业优先，不是通用引擎优先）：

1. **先吃透1个行业**（建材） — 用刘为的案例打磨完整流程
2. **再验证3个相邻行业** — 装修、采购、汽修等"显性省钱型"行业
3. **最后才抽象公共引擎** — 基于验证数据提取通用模板

行业分类（可变现性从高到低）：
- 显性省钱型（建材、采购、装修、汽修）— 最好卖
- 显性增收型（销售、招商、门店运营）— 还可以
- 流程判断型（项目管理、供应链、质检）— 需要包装
- 组织政治型（大公司汇报、跨部门协作）— 最难卖

## Business Model

**前期（验证阶段）**：
- 基础功能免费 — 先帮用户赚到第一笔钱
- 创始人承担 API 成本（单用户约2-3元）

**中期（验证后）**：
- 从成交中抽成，或收结果导向费用
- 增值服务收费：图片生成、视频生成、多平台分发

**后期（规模化）**：
- 订阅制或按次付费

## Validation Metrics

**闲鱼帖子验证（Track 1，进行中）**：
- 帖子已发布，等待48小时客户反馈
- 目标：刘为赚到第一笔知识收入（>=200元）

**产品 MVP 验证（Track 2）**：
- 至少1个非刘为的用户完成全流程
- 推荐的 offer 用户觉得靠谱
- 生成的文案用户愿意发布

## Failure Criteria

- 闲鱼帖子2周内零咨询 → 需求假设失败
- 刘为提供咨询后客户觉得没用 → 知识变现假设失败
- 客户只问免费问题不愿付费 → 省钱需求 ≠ 付费需求
- 用户完成阶段1后不想继续 → 信心重建不够或路径推荐不靠谱
- AI 生成的 offer 用户自己都觉得不靠谱 → AI 质量不足

## Open Questions

1. 信心重建的最佳节奏是什么？多久给一次正向反馈？
2. Case-first 方法对"沉默型"用户是否有效？
3. 从1个行业到3个行业，需要多少模板化工作？
4. 防过度鼓励的边界在哪？怎么判断"该鼓励"还是"该追问"？

## Cross-Model Perspectives

### Codex (GPT-5.4) 的关键贡献：

1. **Case-first 方法** — 用户不会描述自己，但会描述具体事。从案例提取比自我评估更有效。
2. **证据题 vs 评价题** — 不要问"你的优势是什么"，要问"你帮谁省过钱"。
3. **信心重建需要外部证据** — AI 的鼓励不够，市场反馈才是真正的信心来源。
4. **防过度鼓励** — 有证据才强化，不把普通经历硬包装成稀缺能力。
5. **行业优先** — 先做1个行业的模板，再扩展，不要先做通用引擎。
6. **MVP 应该极小** — 但创始人选择保留4个skill（后两个简化版），保证用户体验完整。

### Claude 的关键贡献：

1. **心理重建过程** — 不能跳过"认识自己"直接到"怎么卖"，用户接不住。
2. **Skill 引擎架构** — 产品像 gstack 的 skill 系统，模块化、可扩展。
3. **信心重建是核心** — 不是附加功能，贯穿全产品。
4. **选项+自由补充** — 降低思考负担，同时保留表达空间。

### 分歧与综合：
- Codex 建议 MVP 只做2个流程，创始人坚持保留4个skill（后两个简化）
- Codex 建议 Case-first，创始人吸收但保留分阶段旅程框架
- 两者都同意：先验证、后扩展
