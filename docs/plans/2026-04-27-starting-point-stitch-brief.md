# Starting Point (启点) — Stitch 设计 Brief

> 这是一个给 AI 设计工具的创意简报。定义了"做什么、给谁用、什么感觉"，但把"怎么做"交给你。

---

## 1. 产品是什么

**Starting Point (启点)** (启点)

一句话：帮中年失业者把行业经验变成真金白银的 AI 聊天工具。

目标用户：40-55 岁中国人，干了 10-20 年行业（建材、餐饮、零售等），因行业衰退失业，正在送外卖或跑滴滴维持生计。他们不是没本事，是不知道怎么把经验卖出去。

典型用户画像：刘为，44 岁，建材行业 20 年，瓷砖出厂价门儿清，但行业崩了，只能送外卖。有老有小，信心在谷底。

产品目标：帮他赚到第一笔来自"知识"的钱，不是靠体力，是靠他知道的东西。

---

## 2. 用户心理 (设计必须回应)

这是最关键的部分。用户的心理状态决定了一切设计决策：

**信心崩塌** — 他们投了几十份简历没回音，觉得自己没用了。产品不能让他们觉得"又要学一个新东西"。每一步都要让他们感觉"这个我知道"、"这个我行"。

**时间焦虑** — 他们需要赚钱，不是学习。产品不能有冗长的引导、教程、onboarding。第一个问题就应该让他们开始讲述自己的故事。

**技术恐惧** — 他们不用复杂工具。产品必须像聊天一样简单。不需要学习成本。

**防御心理** — 被拒绝太多次，他们本能地怀疑一切"帮你"的东西。产品不能有居高临下的语气。不是"我来帮你"，是"你的经验很值钱，我们一起看看怎么变现"。

**渴望认同** — 他们最需要的不是信息，是有人说"你懂的东西有人愿意付钱"。产品的每一个正向反馈都必须真诚、具体、不空洞。

---

## 3. 品牌性格

如果这个产品是一个人，它是：

- 安静的，不是聒噪的
- 懂行的，不是高高在上的
- 靠谱的，不是花哨的
- 黑夜里的提灯人，不是舞台上的聚光灯

关键词：克制、沉稳、温暖、有力量。

**不是**：活泼的、年轻的、炫酷的、卡通的。

---

## 4. 视觉方向

**暗色 + 金色**

核心隐喻：经验是暗夜里的金子。打开这个产品，像打开一个精致的盒子。

你需要决定：

- 具体的暗色方案（接近纯黑？深蓝灰？深暖灰？）
- 金色的色调（暖金？玫瑰金？淡金？）和用量
- 金色出现在哪里（CTA 按钮？进度条？关键标签？还是只在最重要的地方用一次？）
- 背景是否需要微妙的纹理或渐变

约束：

- 金色是稀缺资源。用得越少越珍贵。每个屏幕的金色元素不超过 2-3 个。
- 没有浅色模式。只有暗色。
- 没有 box-shadow。暗色界面用背景色差异做层次，不用阴影。
- 图片和插画越少越好。这是文字驱动的产品，不是视觉驱动的产品。

## 5. 核心交互模式：对话

这不是一个 dashboard，不是一个工具箱。它是一个**聊天界面**。

用户 80% 的时间在和一个 AI 对话。AI 问问题，用户回答（选选项或打字）。AI 根据回答给予正向反馈，然后问下一个问题。

对话流程的关键特征：

1. **一次一个问题** — 不给用户看表单。一个问题，等回答，再问下一个。
2. **选项 + 自由输入** — 每个问题有几个预设选项（降低输入门槛），也允许自由输入（让愿意多说的用户展开）。
3. **正向反馈穿插** — AI 不只是问下一个问题。在关键节点，它会暂停说"你刚才说的那个很厉害"或"有人愿意为这个付钱"。这些反馈必须视觉上突出，但不能打断对话节奏。
4. **步骤有终点** — 对话不是无限的。每 6-8 个问题后，输出一个具体的成果（能力清单、变现方案等），让用户有"做到了"的感觉。

---

## 6. 用户旅程 (4 个阶段)

整个产品是 4 段对话，顺序进行：

### 阶段 1: 认识自己 (Self Discovery)

8 个问题，帮用户梳理自己知道什么、擅长什么。

问题示例：
- 你在哪个行业干了多少年？
- 讲一个你帮别人省钱或避坑的真实例子
- 你知道哪些"行内人觉得正常，外行根本不知道"的信息？

输出物：**能力清单** — 列出用户拥有的可变现知识和技能，每项附上估值范围。

心理目标："原来我懂这么多。"

### 阶段 2: 规划路径 (Plan Path)

6 个问题，基于能力清单，推荐变现方向。

问题示例：
- 你最急的是几天内见到钱，还是先做口碑？
- 你愿意和陌生人聊天还是只通过文字？
- 你每周能投入几小时？

输出物：**变现方案** — 3 条具体路径，每条包含：做什么、卖给谁、怎么定价、第一步做什么。

心理目标："这个我好像能做。"

### 阶段 3: 开张行动 (Take Action)

帮用户执行第一个方案。生成发布内容（如闲鱼帖子文案、小红书笔记等）。

输出物：**可发布的文案** — 适配目标平台的帖子内容，用户直接复制发布。

心理目标："我马上就能开始。"

### 阶段 4: 卡住了？(Troubleshoot)

用户遇到问题时提供帮助。不是必须按顺序，随时可以进入。

心理目标："没关系，我们换个方式。"

## 7. 需要设计的页面

### 7.1 首页 (Landing)

一个屏幕，一个故事。

必须包含：
- 产品名称和一句话价值主张（让用户觉得"这是给我的"）
- 开始按钮
- 流程预览（让用户知道大概几步，不会掉进无底洞）

不需要：
- 用户评价/案例（MVP 没有）
- 功能介绍（第一个问题就是最好的介绍）
- 注册/登录（直接开始）

情绪：安静、有力量。"你来了，坐，聊聊。"

### 7.2 聊天界面 (Chat)

产品的核心。80% 的使用时间在这里。

必须包含：
- 当前阶段名称和进度（用户需要知道自己在哪、还有多久）
- AI 消息区域
- 用户回复区域（选项按钮 + 自由输入）
- 正向反馈的展示方式（不能和普通消息混在一起）
- 成果卡片的展示方式（阶段完成时，成果需要视觉上"有分量"）
- 阶段导航（用户能知道自己完成了什么、接下来做什么）

交互状态需要考虑：
- AI 思考中（等待反馈）
- 用户输入中
- 正向反馈出现时
- 阶段完成、成果出现时
- 网络错误

### 7.3 成果总览 (Results)

用户完成一个阶段后，可以随时回顾自己的成果。

必须包含：
- 阶段完成状态（哪些做了、哪些还没做）
- 每个阶段的产出物（能力清单、变现方案、发布文案）
- 继续推进的入口

### 7.4 侧边导航 (Drawer/Menu)

让用户随时看到全局进度，跳转到不同阶段。

必须包含：
- 4 个阶段的列表
- 每个阶段的状态（完成/进行中/未解锁）

## 9. API 数据结构

Stitch 需要理解后端 API 返回什么数据，以便设计正确的 UI 组件。

### 核心请求

```
POST /api/chat
{
  "user_id": "string",
  "message": "string",           // 用户输入的文字
  "selected_option": "string?"   // 选中的选项值（二选一）
}
```

### 核心响应

```json
{
  "message": {
    "role": "assistant",
    "content": "你在哪个行业干了多少年？",
    "options": [
      {"label": "建材/装修", "value": "construction"},
      {"label": "餐饮/食品", "value": "food"},
      {"label": "零售/电商", "value": "retail"}
    ],
    "allow_free_text": true,
    "step_id": "industry",
    "confidence_boost": null
  },
  "progress": 0.125,
  "deliverable": null,
  "skill_completed": false
}
```

关键字段说明：

- `message.options` — 预设选项列表。如果有选项，需要显示选项按钮。
- `message.allow_free_text` — 如果为 true，用户也可以自己打字。
- `message.confidence_boost` — 如果非 null，这是一条正向鼓励，需要视觉上区别于普通消息。
- `progress` — 0 到 1 的进度值。
- `deliverable` — 阶段完成时的成果数据。
- `skill_completed` — 当前阶段是否完成。

### 成果数据结构

**能力清单 (Self Discovery 输出)**：

```json
{
  "capabilities": [
    {
      "name": "装修报价单审核与砍价",
      "description": "帮人看报价单，告诉他哪里被坑了",
      "evidence": "帮朋友省了3万块",
      "estimated_value": "500-2000元/单"
    }
  ],
  "confidence_level": "medium"
}
```

**变现方案 (Plan Path 输出)**：

```json
[
  {
    "name": "装修建材线上咨询",
    "target_customer": "正在装修的业主",
    "pricing": "首次免费，后续99元/次",
    "delivery_format": "微信文字/语音",
    "why_fits": "20年建材经验，对价格了如指掌",
    "first_step_7days": "在闲鱼发一篇避坑帖"
  }
]
```

**发布内容 (Take Action 输出)**：

```json
{
  "platform": "闲鱼",
  "title": "装修建材帮你省钱",
  "body": "10年建材老兵帮你审清单、找平替、避大坑...",
  "tags": ["装修避坑", "建材咨询"],
  "publish_tips": ["带真实案例图", "价格标清楚"]
}
```

### 其他 API

```
POST /api/back/{user_id}/{step_id}  — 回退到某一步
GET  /api/state/{user_id}           — 获取用户当前状态
```

## 10. 技术约束

- **平台**：Web 优先，移动端优先设计。后续可能做微信小程序，所以组件需要可复用。
- **浏览器**：Safari (iOS)、Chrome (Android/PC)。不需要兼容 IE。
- **字体**：中文产品，PingFang SC 是主要字体。确保中文字体回退链正确。
- **最小触控区域**：44x44px（Apple 标准）
- **底部安全区**：iPhone 底部 34px
- **移动端输入框字体**：至少 16px（防止 iOS Safari 自动缩放页面）
- **语言**：界面文案全部中文。`lang="zh-CN"`
- **后端已就绪**：FastAPI API 已经可以调用，前端只需要实现 UI 层和 API 对接。

## 最后

这个产品的成功标准不是"看起来多酷"，而是：**一个信心崩塌的中年人，用完之后觉得自己真的能行。**

每做一个设计决定，问自己：这个会让刘为觉得更自信，还是更焦虑？如果是后者，删掉它。
