# Starting Point V2: 全流程引导工具设计

> 目标：帮40-55岁失业中年人，在30天内完成从"我什么都不会"到"赚到第一笔钱"的全过程。
> 验证用户：44岁建材行业小老板，行业塌了，投简历无回复，试过滴滴/外卖干不过年轻人。

## 核心理念

启点V1是"帮你认识自己的工具"。V2是"陪你赚到第一块钱的教练"。

关键原则：
1. **不给过高期望。** 目标用户不是创业者，是"想继续当牛马但没人要"的人。期望值管理是生死线。
2. **每一步给可直接用的东西。** 不是建议，是话术、模板、脚本、checklist。
3. **情绪支持比工具重要。** 失业中年人最大的敌人是放弃，不是缺少工具。
4. **不同用户不同策略。** 先评估再行动，不搞一刀切。
5. **每一步都能回退。** 用户填错、想重新思考、暂停休息后继续，都要支持。

## 用户旅程（6阶段）

```
阶段0: 起跑评估
  → 评估数字能力、心理准备、时间投入、经济压力
  → 产出：用户画像标签 + 个性化策略 + 内容节奏

阶段1: 发现金矿（增强现有 Self Discovery）
  → 用具体故事提取可定价资产
  → 产出：可定价资产清单（每项资产带市场参考价）

阶段2: 包装产品（新增核心）
  → 资产 → 服务产品卡片
  → 产出：服务名称 + 定价策略 + 服务流程 + 交付物清单

阶段3: 找到客户（重设计，30天内容计划）
  → 按周制定内容计划，每天一条，持续30天
  → 产出：30条内容脚本 + 每周调整建议 + 情绪支持

阶段4: 完成首单（全新）
  → 客户来了之后的完整流程
  → 产出：沟通模板 + 报价公式 + 收钱方式 + 交付checklist

阶段5: 转起来（新增）
  → 首单后持续经营
  → 产出：口碑素材化 + 定价调整 + 转介绍机制 + 复购设计
```

## 阶段0：起跑评估

### 评估维度（4个问题）

1. **数字能力：** 你平时用手机做什么？（只打电话微信 / 刷抖音小红书 / 发过视频或帖子）
2. **心理准备：** 你现在最想的是什么？（找到一份工作 / 试试自己干 / 不确定）
3. **时间投入：** 你每天能花多少时间在这件事上？（1小时内 / 1-3小时 / 3小时以上）
4. **经济压力：** 你现在的经济状况？（还有3个月以上缓冲 / 1-3个月 / 一个月内很紧张）

### 评估产出

用户画像标签，示例：
- "数字新手 + 心理犹豫 + 时间充裕 + 高压力"
- "有基础 + 愿意试 + 时间一般 + 低压力"

### 个性化策略

| 用户类型 | 内容节奏 | 首次目标 | 期望管理 |
|---------|---------|---------|---------|
| 数字新手 + 高压力 | 极慢，先学工具 | 注册一个小红书账号 | "今天只做一件事" |
| 数字新手 + 低压力 | 慢，每周3条 | 发10条内容 | "先让10条发出去，不管效果" |
| 有基础 + 高压力 | 中速，每天1条 | 获得第一个咨询 | "目标是被人问到，不是成交" |
| 有基础 + 低压力 | 正常，每天2-3条 | 30天内完成首单 | "30条是起点，坚持一个月" |

### AI模型推荐

- 主力：DeepSeek V4（中文理解强，便宜）
- 平替：通义千问（阿里，免费额度大）/ 豆包（字节，免费）

## 阶段1：发现金矿（增强）

### 与V1的区别

- 保留现有的案例引导式提问
- 产出从"资产清单"变成"可定价资产清单"
- 每项资产后面附带市场参考价区间
- 根据阶段0的用户画像调整提问语气和节奏

### 产出格式

```
你的可定价资产：
1. 建材选材能力 — 市场参考：单次咨询 200-500元
   证据：你在建材行业15年，知道哪里买便宜、哪些品牌性价比高
2. 报价审核能力 — 市场参考：单次审核 300-800元
   证据：你帮朋友看过装修报价，找出过3处水分
3. 验收指导能力 — 市场参考：单次上门 500-1500元
   证据：你知道装修完要看哪些地方、怎么判断施工质量
```

## 阶段2：包装产品

### 产出：服务产品卡片

```yaml
服务名称: "装修避坑顾问"
一句话定位: "15年建材老兵，帮你省下装修中的冤枉钱"
目标客户: "正在装修或准备装修的业主"
定价策略:
  体验价: 99元/次（30分钟电话咨询）
  正式价: 299-599元/次
  套餐价: 999元（全程陪跑，从选材到验收）
服务流程:
  1. 客户发来装修方案/报价单
  2. 你标注需要关注的点
  3. 30分钟电话/视频沟通
  4. 出具文字版建议清单
交付物: "避坑建议清单 + 推荐品牌/渠道列表"
```

### AI工具推荐

- 服务设计：DeepSeek V4 / Kimi（长上下文好）
- 同行参考价：内置行业价格数据库（人工整理）
- 图像（产品卡片封面）：通义万相（免费）/ Canva模板

## 阶段3：找到客户（30天内容计划）

### 核心原则

5条内容不够。新账号冷启动需要30条起步，持续发。

不给"你会很快赚到钱"的暗示。里程碑不是"成交"，而是：
- 第1周：发出去了
- 第2周：有人看了
- 第3周：有人问了
- 第4周：有人付钱了

### 第1周：试水期（7条内容）

- 目标：熟悉发内容的流程，不追求效果
- 内容类型交替：2条经验分享 + 2条避坑指南 + 2条故事 + 1条互动问答
- 情绪管理："第一周不求浏览量，只求你完成了。发出去就是胜利。"

### 第2周：找感觉期（7条内容）

- 目标：观察哪类内容有人看，调整方向
- 根据第1周数据反馈调整内容方向
- 如果没有数据：提醒"新账号2-4周才能看到效果，继续发"
- 情绪管理："还没效果是正常的，90%的人在这一周放弃。你坚持了就是前10%。"

### 第3周：突破期（8条内容）

- 目标：获得第一个咨询/私信
- 增加互动引导："评论区留言你的装修问题"
- 获得第一个咨询 → 进入阶段4
- 没有咨询 → 系统生成"诊断报告"分析可能原因
- 情绪管理："有人开始关注你了，哪怕只有一个。方向对了。"

### 第4周：收获期（8条内容）

- 目标：稳定获得咨询，向成交转化
- 有咨询没成交 → 阶段4强化沟通能力
- 有成交 → 阶段5
- 30天总复盘

### 渠道策略

**抖音：**
- 内容形式：真人出镜口播 / 图文配音
- 工具：剪映（免费，抖音官方）剪辑和加字幕
- 脚本生成：DeepSeek V4 或 豆包（字节系对抖音内容理解更好）

**小红书：**
- 内容形式：图文笔记
- 工具：Canva（免费版）做封面图
- 内容生成：DeepSeek V4 或 Kimi
- 图像：通义万相（免费）/ Midjourney（付费）

**朋友圈：**
- 一周内容计划：7条，交替专业内容和生活内容
- 不让人反感地发广告的方式

### 每周系统产出

- 本周7-8条内容的具体脚本/文案
- 下周内容方向建议
- 情绪支持话术（基于用户实际进度）
- 进度可视化（发了多少条 / 浏览量 / 咨询数）

## 阶段4：完成首单

### 沟通模板（3种场景）

**问价的客户：**
- "感谢咨询！我提供30分钟免费电话咨询，先了解你的具体需求，然后给你一个清晰的报价。方便留个电话吗？"

**问服务的客户：**
- "我的服务是帮你从选材到验收全程把关，避免装修中的常见坑。你现在是哪个阶段？"

**犹豫的客户：**
- "理解你的顾虑。要不先看看我发的内容，觉得靠谱再联系我？我的[小红书/抖音]主页有很多真实案例。"

### 报价公式

- 基础价 = 阶段2中定价策略的正式价
- 根据客户需求复杂度调整（±30%）
- 首单优惠：8折，但不说"打折"，说"首次合作体验价"

### 收钱方式

- 微信转账（最简单，无担保）
- 闲鱼担保交易（通过闲鱼来的客户）
- 建议：先收50%定金，交付后收尾款

### 交付checklist

根据阶段2的服务流程，生成可打勾的清单。完成一项打一项勾。全部完成 = 服务结束。

### 交付后动作

- 引导客户反馈："麻烦给我一个评价，对你来说是举手之劳，对我来说很重要"
- 客户好评 → 变成内容素材（阶段5）

## 阶段5：转起来

### 四个动作

1. **口碑素材化：** 客户好评变成小红书/抖音的真实案例内容
2. **定价调整：** 完成3单后考虑涨价（每次涨20%）
3. **转介绍：** 老客户带新客户，给老客户一次免费咨询
4. **复购设计：** 一次性服务 → 包月/包季陪跑

## 数据流

### 阶段间数据传递

```
阶段0 → 全程：用户画像标签 + 个性化策略 + 内容节奏 + 期望管理策略
阶段1 → 阶段2：行业 + 可定价资产清单 + 信心分数
阶段2 → 阶段3：服务名称 + 目标客户 + 定价 + 服务流程
阶段3 → 阶段4：渠道 + 内容素材 + 客户来源
阶段4 → 阶段5：首单记录 + 客户反馈 + 实际成交价
```

### 用户状态模型

```python
class UserState:
    user_id: str
    current_phase: int  # 0-5
    current_step: str

    # 阶段0产出
    assessment: UserAssessment  # 数字能力 + 心理 + 时间 + 压力

    # 各阶段产出
    phase_results: dict[int, PhaseResult]

    # 贯穿全程
    industry: str
    confidence_score: float

    # 阶段3专用：30天内容计划进度
    content_plan: ContentPlan | None

    created_at: datetime
    updated_at: datetime
```

### 回退机制

- 任意步骤可回退到上一步
- 回退不删除数据，进入编辑状态
- 阶段1数据变更 → 提示"服务产品和渠道内容需要更新"
- 阶段3可暂停（用户累了），系统记住进度，回来时从断点继续
- 阶段0可重新评估（用户情况变化），重新评估后调整后续策略
- 阶段2-5产出有版本记录，回退时可以恢复之前的版本

## 降级方案（用户无AI时）

如果用户没有AI工具，内置模板系统仍然能工作：

1. **行业模板库：** 建材、餐饮、零售等常见行业的预置服务包
2. **定价参考表：** 不用AI，直接给区间
3. **内容模板：** 填空式文案模板，用户只需填自己的信息
4. **话术模板：** 常见沟通场景的固定话术

## 技术架构

保持现有的 FastAPI + SkillRegistry + SkillRunner + StateManager 架构。

### 改动范围

1. **新增Skill：** AssessmentSkill（阶段0），ProductPackagingSkill（阶段2），FirstDealSkill（阶段4），GrowthSkill（阶段5）
2. **重写Skill：** TakeActionSkill → CustomerAcquisitionSkill（阶段3，30天计划）
3. **增强Skill：** SelfDiscoverySkill（阶段1，增加定价参考）
4. **状态管理：** 扩展StateManager支持更复杂的数据结构、版本记录、暂停/恢复
5. **新增API：** 内容计划进度追踪、暂停/恢复、重新评估
6. **新增UI页面：** 评估页面、产品卡片页面、30天内容计划页面、首单手册页面、增长计划页面
7. **进度条重设计：** 从线性步数变成阶段展开/折叠

llm_profiles = {
