# Design: BuilderPulse-Inspired AI Diagnosis Enhancement

Generated by /brainstorming on 2026-04-30

Status: APPROVED

## Background

BuilderPulse (github.com/BuilderPulse/BuilderPulse) is an AI-powered daily intelligence product for indie hackers. It answers 20 fixed questions daily from 10+ data sources, then delivers actionable "2-hour build" recommendations.

Three BuilderPulse patterns apply to Starting Point:

1. **Fixed question framework** — structured questions produce better signal than open-ended chat
2. **Market signal validation** — external evidence that "someone is already looking for this"
3. **Actionable next step** — not a plan, but a specific thing to do today

## What Changes

Three enhancements to Starting Point's AI diagnosis pipeline, ordered by implementation priority.

---

## Phase 1: Market Validation Questions in SelfDiscoverySkill

**What:** Add 3 new questions to `self_discovery.py` after the existing 8 steps.

**New questions:**

| Step ID | Question | BuilderPulse inspiration | Purpose |
|---------|----------|--------------------------|---------|
| `content_search` | "你在抖音或小红书上搜过自己行业的什么内容？" | Search spike analysis | Identify user's own demand signals |
| `organic_inquiry` | "最近有没有人主动找你帮忙或咨询？因为什么事？" | Solo-founder product discovery | Detect pre-existing "payment" (attention/favors) |
| `shared_pain` | "你觉得你行业里什么最坑？你朋友也这么觉得吗？" | Developer complaint analysis | Shared pain = scalable opportunity |

**Files changed:**
- `src/starting_point/skills/self_discovery.py` — add 3 Step entries to `steps` list
- `src/starting_point/llm/prompts.py` — update `EXTRACTION_TEMPLATE` to incorporate market signal answers
- No frontend changes needed (existing step UI handles new steps)

**Risk:** Minimal. Purely additive. Existing 8 questions and their processing logic unchanged.

## Phase 2: 7-Day Daily Task Cards

**What:** Replace the current `take_action.py` output with a 7-day daily task plan. Each day has one concrete, low-barrier action.

**Output format per task:**
```json
{
  "day": 1,
  "day_label": "今天",
  "task": "在闲鱼发一条'装修建材避坑咨询，首次免费'",
  "platform": "闲鱼",
  "estimated_time": "30分钟",
  "why": "你上次说的瓷砖选品经验，正是装修业主最需要的",
  "success_signal": "有人来问'我家想铺地砖，怎么选'"
}
```

**Files changed:**
- `src/starting_point/llm/prompts.py` — add `DAILY_TASKS_TEMPLATE` prompt
- `src/starting_point/skills/take_action.py` — `generate_output` produces 7 daily tasks instead of generic action plan
- `static/js/app.js` — new "daily task card" display mode with checkbox per task
- `static/js/store.js` — persist task completion state in localStorage

**Design decisions:**
- Tasks are generated once when user reaches the action phase (not re-generated daily)
- User can check off completed tasks; state persists in localStorage
- LLM considers: user's asset map, time commitment, digital literacy, financial pressure
- Each task references specific evidence from the user's self-discovery answers

## Phase 3: Industry Opportunity Radar (MVP: LLM Knowledge-Driven)

**What:** A new `MarketSignalSkill` that tells the user "people are already looking for your expertise."

**MVP approach (no external API):**
- User's industry + asset map → LLM generates analysis of existing market demand
- Output: "In your industry, these monetization paths already have buyers:"
  - What's already being sold on 闲鱼/小红书 in this space
  - Which content topics get engagement
  - Price ranges people charge
  - Gaps where your specific experience stands out

**Future enhancement (post-MVP):**
- Integrate search APIs (闲鱼, 小红书) for real-time signal data
- Cross-reference user's specific capabilities with trending content

**Files changed:**
- `src/starting_point/skills/market_signal.py` — new skill
- `src/starting_point/llm/prompts.py` — add `MARKET_SIGNAL_TEMPLATE`
- `static/js/app.js` — display market signal results
- `src/starting_point/engine/runner.py` — register new skill in pipeline

**Risk:** LLM-generated market signals are informed estimates, not real data. The output must clearly frame them as "based on general market knowledge" rather than verified facts.

| Phase | Duration | Dependencies |

| Phase 1: Market validation questions | 1-2 days | None |

## Success Criteria

- Phase 1: New questions produce answers that improve LLM asset extraction quality
- Phase 2: Users can see and check off daily tasks; task content references their specific answers
- Phase 3: Market signal output gives at least one insight the user didn't know before
